How Accessible Name and Description Computation is Used

The Computation Process

The specification defines a precise algorithm that browsers and assistive technologies follow to determine an element's accessible name:

  1. Step 1: Check for explicit labels (aria-labelledby, aria-label)
  2. Step 2: Check native HTML semantics (label element, alt attribute)
  3. Step 3: Check content (text nodes, placeholder)
  4. Step 4: Fallback mechanisms (title attribute, etc.)

Practical Implementation Examples

Priority Order in Practice:

<button aria-label="Close" title="Close window">X</button>
// Accessible name: "Close" (aria-label wins)

<img src="icon.png" alt="Settings" title="Configuration">
// Accessible name: "Settings" (alt attribute wins)

<label for="name">Full Name</label>
<input id="name" aria-label="Your name">
// Accessible name: "Your name" (aria-label overrides label)

Browser and AT Integration

Browsers implement the algorithm and expose the computed name through accessibility APIs:

  • Windows: Exposed through MSAA/IAccessible2 and UI Automation
  • macOS: Exposed through NSAccessibility
  • Linux: Exposed through AT-SPI/ATK
